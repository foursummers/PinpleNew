# 拼朋友 Pinple — 产品功能描述文档

> **版本**：v4.0 &nbsp;|&nbsp; **定位**：信任圈社交平台 + 技能共享市场  
> **Slogan**：养孩子要拼，找工作要拼，活下来更要拼。

---

## 一、产品愿景

**拼朋友 (Pinple)** 是一款面向年轻家庭和个体从业者的社交+服务平台。

我们从"家庭管理工具"升级为 **信任圈社交 + 技能共享市场**，核心理念：

- 每个人都在"拼"——拼事业、拼育儿、拼生活
- 通过真实社交关系构建信任网络，解决信息不对称
- 让技能供需双方在信任基础上高效匹配

---

## 二、核心功能模块

### 2.1 家庭管理（保留+增强）

| 功能 | 描述 |
|------|------|
| **多家庭支持** | 一个用户可创建/加入多个家庭空间 |
| **孩子档案** | 姓名、性别、生日、头像、相册、成长记录 |
| **成长时间轴** | 自动+手动事件混合排列，支持公开/私密切换 |
| **日常任务** | 家庭成员协作的任务清单，支持打卡+月历视图 |
| **活动管理** | 家庭活动创建、邀请码分享、好友参与申请 |
| **成员管理** | 家庭成员角色、出生日期、纪念日、大事件倒计时 |
| **IVF 倒计时** | 双胎 IVF 预产期计算（37 周+40 周） |

### 2.2 人脉圈（信任网络）

| 功能 | 描述 |
|------|------|
| **关注/互关** | 单向关注 + 双向互关，互关可查看更多信息 |
| **推荐链** | A 把 B 推荐给 C → C 看到 B 的名片 + A 的推荐理由 + A 的联系方式，形成可追溯信任链 |
| **名片分享** | 将用户名片以卡片形式分享给他人 |
| **共同好友** | 查看与目标用户的共同好友数量和列表 |
| **好友分类** | 亲友 / 同事 / 服务者 / 其他，支持自定义标签 |
| **好友动态** | 查看好友公开的时间轴事件和活动 |

### 2.3 技能市场

| 功能 | 描述 |
|------|------|
| **发布技能** | 用户发布自己擅长的技能/服务：名称、分类、描述、价格区间、服务地点、服务半径 |
| **发布求助** | 用户发布需求：标题、描述、技能标签、预算、地点、紧急程度、截止日期 |
| **智能匹配** | 系统根据地理位置 + 技能标签 + 价格 + 信用分自动推荐匹配 |
| **接单沟通** | 技能提供者查看求助详情，标记"接受"，通过 App 内留言或微信联系 |
| **订单流程** | 匹配 → 接受 → 完成 → 评价，全链路闭环 |
| **分类筛选** | 教育 / 育儿 / 家政 / 技术 / 其他 |

**典型场景**：
- 月嫂 A 在平台发布"月嫂服务"，标明经验、价格、服务区域
- 主家 B 发布"寻找有经验月嫂"求助
- 系统自动匹配 → B 看到 A 的技能卡 + 信用分 + 历史评价 → 联系接单

### 2.4 用户信用体系

| 规则 | 说明 |
|------|------|
| **初始信用分** | 100 分 |
| **完成订单** | +5 分 |
| **获得好评（4-5 星）** | +2 分 |
| **被举报（审核通过）** | -10 分 |
| **信用分 < 20** | 禁止发布技能和求助 |
| **信用分 < 10** | 禁止登录 |

| 功能 | 描述 |
|------|------|
| **举报系统** | 选择原因（不当内容 / 欺诈 / 骚扰 / 其他）+ 描述 + 证据上传 |
| **管理员审核** | 管理员查看举报列表 → 通过/拒绝 → 通过时自动扣信用分 |
| **屏蔽系统** | 一键屏蔽 / 取消屏蔽，被屏蔽用户不可查看你的信息 |
| **评价系统** | 订单完成后互评（1-5 星 + 文字评价） |

### 2.5 评价与口碑

| 功能 | 描述 |
|------|------|
| **星级评价** | 完成匹配后双方可对对方评价（1-5 星） |
| **评价详情** | 评价文字内容 + 评分 + 时间 |
| **评价列表** | 其他用户可查看某人收到的所有评价 |
| **信用联动** | 好评自动加信用分，差评不扣分但可追踪 |

## 三、登录系统

| 方式 | 状态 | 说明 |
|------|------|------|
| **Google OAuth** | ✅ 已实现 | 通过 Manus OAuth 代理，一键 Google 登录 |
| **邮箱密码** | ✅ 已实现 | 注册（邮箱+密码+昵称）+ 登录，SHA-256 加密 |
| **微信登录** | 🔲 占位 | 预留 wechatOpenId 字段和 UI 入口，待对接微信 SDK |

**开屏流程**：
1. 2-3 秒品牌呼吸动效（Lottie 动画 + 渐变背景）
2. 自动检测登录状态
3. 已登录 → 跳转首页；未登录 → 跳转登录页
4. 登录页三选一：Google / 微信 / 邮箱密码

## 七、核心创新点

### 7.1 推荐链可追溯

```
主家 B 推荐月嫂 A 给 主家 C
         ↓
主家 C 看到：
  ├── 月嫂 A 的技能名片
  ├── 主家 B 的推荐理由
  └── 主家 B 的联系方式（可验证）
```

解决传统口碑推荐"口说无凭"的痛点，形成可追溯的信任网络。

### 7.2 技能市场智能匹配

```
用户发布 "寻找钢琴老师"
         ↓
系统匹配维度：
  ├── 地理位置（同城优先）
  ├── 技能标签（钢琴/音乐教育）
  ├── 价格区间（预算匹配）
  └── 信用分（优先高信用）
         ↓
推送匹配结果 → 接单 → 完成 → 评价
```

### 7.3 信用体系闭环

```
注册 (100分)
  → 完成订单 (+5)
  → 获好评 (+2)
  → 被举报 (-10)
  → 低于 20 分 → 禁止发布
  → 低于 10 分 → 禁止登录
```

通过经济激励和惩罚机制，维护平台生态健康。

| 原则 | 说明 |
|------|------|
| **完全保留** | 所有现有用户、家庭、孩子、活动、任务数据无损保留 |
